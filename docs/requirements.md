## 1. プロジェクト概要

### 1.1 背景

仕事やプライベートのタスク・メモ管理を Excel やメモ帳で行っているユーザーが多い。これらのツールは手軽な反面、優先度管理・期限通知・マルチデバイス対応が弱く、情報が散乱しやすい。本サービスは、シンプルな操作で「優先度・期限・完了状態」を一元管理できる Web アプリを提供する。

### 1.2 目的

- Excel・メモ帳からの移行先となるシンプルなタスク管理 SaaS を構築する
- 無料プランで価値を体験させ、有料プランへの自然なアップグレードを促すサブスクリプションモデルを実現する

### 1.3 ターゲットユーザー

| ユーザー層 | 説明 |
|-----------|------|
| 個人ユーザー | 仕事・プライベートのタスクを個人で管理したいフリーランス・会社員 |
| 副業・ソロプレナー | 複数案件を掛け持ちするフリーランサー、タスクが増えてきた個人事業主 |

## 4. 非機能要件

### 4.1 セキュリティ

| ID | 要件 |
|----|------|
| SEC-01 | パスワードは bcrypt でハッシュ化して保存する |
| SEC-02 | JWT の有効期限は 24 時間とし、SECRET_KEY は環境変数で管理する |
| SEC-03 | 他ユーザーのメモにはアクセスできない（user_id によるフィルタリング） |
| SEC-04 | 機密情報（パスワード・シークレットキー）を Git にコミットしない |
| SEC-05 | HTTPS 通信を使用する（本番環境） |

### 4.2 パフォーマンス

| ID | 要件 |
|----|------|
| PERF-01 | メモ一覧の応答時間は 500ms 以内 |
| PERF-02 | ページネーション（デフォルト 50 件）でスケールに対応する |

### 4.3 可用性・運用

| ID | 要件 |
|----|------|
| OPS-01 | Docker Compose でローカル開発環境を再現できる |
| OPS-02 | AWS ECS Fargate で本番運用する |
| OPS-03 | GitHub Actions で CI/CD を構築する |
| OPS-04 | データベースは AWS RDS (PostgreSQL) を使用する（本番環境） |

## 7. API 一覧
